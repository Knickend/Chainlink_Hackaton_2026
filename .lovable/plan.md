

## Security Hardening and Build Fix Plan

### Context

After investigating the codebase, here is the actual security posture:

- The `.env` file is **already in `.gitignore`** (line 35), so it won't be committed going forward.
- The values in `.env` (`VITE_SUPABASE_URL`, `VITE_SUPABASE_PUBLISHABLE_KEY`, `VITE_SUPABASE_PROJECT_ID`) are **publishable/public keys** -- they are the anonymous key designed to be exposed in the frontend. They are NOT secret credentials.
- All actual secrets (service role key, CDP keys, API keys, etc.) are stored securely in Lovable Cloud backend secrets, never in the codebase.
- The `.env` file is auto-generated by Lovable Cloud and should not be manually edited.

### What We Will Do

#### 1. Create Root `.env.example` with Placeholders
Add a root-level `.env.example` file with placeholder values so contributors know what environment variables are needed:

```
# .env.example - Lovable Cloud auto-generates these. For manual/self-hosted setups, fill in your values.
VITE_SUPABASE_PROJECT_ID=your_supabase_project_id
VITE_SUPABASE_PUBLISHABLE_KEY=your_supabase_anon_key
VITE_SUPABASE_URL=https://your-project-id.supabase.co
```

#### 2. Fix Build Errors

**Calendar component** (`src/components/ui/calendar.tsx`): The `react-day-picker` v9 renamed `IconLeft`/`IconRight` to `Chevron`. Update the `components` prop accordingly.

**useDebounce hook** (`src/hooks/useDebounce.ts`): React 19 requires `useRef` to receive an initial value. Change `useRef<ReturnType<typeof setTimeout>>()` to `useRef<ReturnType<typeof setTimeout>>(undefined)`.

#### 3. Git History Note

Removing `.env` from Git history requires running `git filter-repo` on the GitHub repository, which cannot be done from within Lovable. However, since these are **publishable keys** (the anon key is meant to be public and is already visible in the built frontend bundle), this is low risk. If you still want to clean history for hygiene:

1. Clone your repo locally
2. Run: `git filter-repo --path .env --invert-paths`
3. Force-push to GitHub

This is optional given the keys are public by design.
